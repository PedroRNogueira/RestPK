# Assets Structure

## Objetivo
Este documento registra a organização inicial esperada para a área de assets do RestPK.

## Estrutura inicial esperada
Nesta fase, a área de assets deve ser entendida como o espaço futuro para:
- recursos visuais do projeto
- materiais auxiliares de apresentação
- base de apoio visual para cliente e mapas
- organização técnica dos assets do jogo

## Papel dessa estrutura
Esta organização servirá, futuramente, para:
- separar claramente os assets do restante do projeto
- permitir evolução incremental da identidade visual
- facilitar a localização de arquivos importantes
- reduzir ambiguidade durante o bootstrap técnico dos recursos visuais

## O que ainda não entra
- biblioteca visual completa
- sprites finais do projeto
- interface final
- grande volume de recursos gráficos
- organização final dos assets

## Observação
Este documento ainda não representa a estrutura definitiva dos assets.
Ele apenas registra a organização inicial esperada antes da integração técnica real.
