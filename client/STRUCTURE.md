# Client Structure

## Objetivo
Este documento registra a organização inicial esperada para a área do cliente do RestPK.

## Estrutura inicial esperada
Nesta fase, a área do cliente deve ser entendida como o espaço futuro para:
- base principal do cliente
- arquivos de interface visual
- recursos ligados à apresentação do jogo
- organização técnica da lógica de renderização

## Papel dessa estrutura
Esta organização servirá, futuramente, para:
- separar claramente a parte cliente do restante do projeto
- permitir evolução incremental da interface do jogo
- facilitar a localização de arquivos importantes
- reduzir ambiguidade durante o bootstrap técnico do cliente

## O que ainda não entra
- estrutura final importada do OTClient mehah / OpenTibiaBR
- refatoração de arquivos reais
- interface final
- configuração final de execução
- customizações profundas

## Observação
Este documento ainda não representa a estrutura definitiva do cliente.
Ele apenas registra a organização inicial esperada antes da integração técnica real.
