# Client Bootstrap

## Objetivo
Este documento registra a preparação mínima esperada para a base do cliente do RestPK.

## Cliente planejado
O cliente planejado do projeto é o **OTClient mehah / OpenTibiaBR**.

## Função desta etapa
Nesta etapa, o foco ainda não é implementar gameplay completo.
O foco é apenas preparar o entendimento técnico inicial da base do cliente.

## O que deve ser definido nesta fase
- papel do OTClient dentro do projeto
- função do cliente na apresentação visual do jogo
- relação do cliente com servidor, mapa e interface
- estrutura inicial onde a base do cliente ficará no repositório

## O que ainda não entra
- customizações visuais profundas
- interface final do projeto
- sistemas avançados de interação
- polimento gráfico
- conteúdo jogável completo

## Resultado esperado
Ao final desta etapa, o projeto deve ter uma referência documental clara sobre o papel do cliente e sobre o que será necessário para preparar sua base futuramente.

## Observação
Este documento ainda não é um guia de instalação.
Ele serve apenas para alinhar o bootstrap inicial do cliente dentro do projeto.
