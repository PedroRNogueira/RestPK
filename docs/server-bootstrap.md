# Server Bootstrap

## Objetivo
Este documento registra a preparação mínima esperada para a base do servidor do RestPK.

## Servidor planejado
O servidor planejado do projeto é o **Canary**.

## Função desta etapa
Nesta etapa, o foco ainda não é implementar gameplay.
O foco é apenas preparar o entendimento técnico inicial da base do servidor.

## O que deve ser definido nesta fase
- papel do Canary dentro do projeto
- dependências gerais esperadas para trabalhar com o servidor
- estrutura inicial onde a base do servidor ficará no repositório
- relação do servidor com scripts e lógica de jogo

## O que ainda não entra
- instalação completa do servidor
- customizações profundas no código-fonte
- sistemas de batalha
- economia do jogo
- conteúdo jogável completo

## Resultado esperado
Ao final desta etapa, o projeto deve ter uma referência documental clara sobre o papel do servidor e sobre o que será necessário para preparar sua base futuramente.

## Observação
Este documento ainda não é um guia de instalação.
Ele serve apenas para alinhar o bootstrap inicial do servidor dentro do projeto.
