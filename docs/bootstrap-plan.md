# Bootstrap Plan

## Objetivo
Este documento define a ordem inicial de preparação técnica do RestPK para evitar implementação desorganizada da stack.

## Etapa 1: confirmar base documental
Antes de qualquer integração técnica, o projeto deve manter consistência entre:
- README
- roadmap
- setup
- definição da stack OT
- regras do agente

## Etapa 2: preparar base do servidor
O primeiro componente técnico a receber atenção deve ser o servidor.

Objetivo desta etapa:
- documentar a preparação mínima do Canary
- definir o que será necessário para rodar a base do servidor
- registrar dependências e pontos de atenção

## Etapa 3: preparar base do cliente
Depois da base do servidor, o foco deve ir para o cliente.

Objetivo desta etapa:
- documentar a preparação mínima do OTClient
- registrar o papel do cliente no fluxo inicial
- alinhar o que dependerá do servidor

## Etapa 4: preparar base do mapa
Depois de servidor e cliente, o foco deve ir para o mapa.

Objetivo desta etapa:
- documentar a preparação mínima do RME
- registrar como o mapa entra no fluxo do projeto
- alinhar o papel do conteúdo espacial inicial

## Etapa 5: alinhar primeiro loop técnico
Com servidor, cliente e mapa minimamente planejados, o projeto poderá definir o primeiro loop técnico real.

Esse primeiro loop deverá considerar:
- entrada no jogo
- movimentação básica
- primeiro mapa funcional
- primeira interação mínima

## O que ainda não entra
- conteúdo extenso
- grande quantidade de criaturas
- sistemas avançados
- polimento visual
- expansão de mundo

## Observação
Este plano ainda não representa instalação completa nem integração final da stack.
Ele apenas organiza a ordem correta de avanço técnico do projeto.
