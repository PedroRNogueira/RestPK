# Implementation Order

## Objetivo
Este documento registra a ordem esperada de avanço técnico do RestPK após a fase documental inicial.

## Ordem definida
A ordem inicial de avanço do projeto deve ser:

1. servidor
2. cliente
3. mapas
4. assets
5. tools de suporte

## Motivo da ordem
Essa ordem existe para:
- reduzir ambiguidade de execução
- evitar começar por conteúdo sem base técnica
- manter o núcleo do projeto organizado
- permitir evolução incremental com menor risco

## Interpretação prática
- o servidor deve receber atenção primeiro
- o cliente deve ser organizado depois da base do servidor
- os mapas devem ser tratados após servidor e cliente
- os assets devem acompanhar a evolução da estrutura principal
- os tools devem servir como suporte, não como ponto de partida do projeto

## O que ainda não entra
- importação completa de bases externas
- implementação de gameplay completo
- conteúdo extenso
- polimento final
- expansão grande de escopo

## Observação
Este documento ainda não representa cronograma fechado.
Ele apenas registra a ordem lógica de implementação para evitar desorganização no avanço técnico do projeto.
