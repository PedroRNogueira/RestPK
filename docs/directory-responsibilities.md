# Directory Responsibilities

## Objetivo
Este documento registra a responsabilidade principal de cada diretório base do projeto RestPK.

## Diretório `server/`
Responsável pela futura base do servidor do projeto.
Deve concentrar a lógica principal do jogo, scripts e organização técnica da parte servidor.

## Diretório `client/`
Responsável pela futura base do cliente do projeto.
Deve concentrar a apresentação visual, interface e organização técnica da experiência do jogador.

## Diretório `map/`
Responsável pela futura base de mapas do projeto.
Deve concentrar arquivos de mapa, estrutura espacial e organização técnica das áreas jogáveis.

## Diretório `assets/`
Responsável pelos recursos visuais e auxiliares do projeto.
Deve concentrar materiais de apoio visual e organização técnica dos assets.

## Diretório `tools/`
Responsável por scripts e utilitários auxiliares.
Deve concentrar recursos de suporte ao fluxo de trabalho e ao desenvolvimento.

## Observação
Este documento existe para reduzir ambiguidade sobre o papel de cada diretório antes da integração técnica real do projeto.
